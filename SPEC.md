# Night Drive — специфікація гри

3D-аркада-ухилянка у нічному місті. Гравець керує авто, оминає трафік,
проходить рівні. Працює як Telegram Mini App і як звичайна веб-сторінка.

Технічно: Three.js (WebGL) + TypeScript + Vite. Уся геометрія процедурна,
без зовнішніх 3D-ассетів. Сторінка: `three.html` → `src/three3d/main3d.ts`.

## Ігровий цикл

1. **Меню** — гравець обирає авто та місто, тисне «ГРАТИ».
2. **Гра** — авто їде вперед автоматично; гравець кермує вліво/вправо,
   оминаючи зустрічні машини. Дистанція зростає.
3. **Зіткнення** — удар по машині забирає життя, коротка невразливість.
   При 0 життів — екран «Аварія».
4. **Рівень пройдено** — після досягнення цільової дистанції рівня
   показується тост, складність зростає, гра триває.

## Керування

- Тягни пальцем (або мишею) ліворуч/праворуч — авто рухається по X.
- Авто завжди їде вперед; швидкість залежить від рівня.
- Кермування активне лише у стані `playing`.

## Стани (state machine)

`menu` → `playing` → (`crashed` | `gameover`) → `menu`/`playing`

- **menu**: видно оверлей меню; HUD сховано; рендер сцени триває (фон).
- **playing**: HUD видно; меню/аварія сховані; працює спавн трафіку,
  зіткнення, нарахування дистанції, перевірка рівня.
- **crashed (life lost)**: коротка пауза-невразливість (≈1.2 с), блимання
  авто, трафік поряд відкидається; потім знову `playing`.
- **gameover**: оверлей «Аварія» з рахунком; кнопки «Ще раз» / «Меню».

## Трафік і зіткнення

- Дорога має дискретні смуги: `LANES = [-2.4, -0.8, 0.8, 2.4]`.
- Машини спавняться попереду (`z ≈ -90`) у вільній смузі та рухаються
  назустріч (нетто-швидкість = швидкість гравця − власна швидкість трафіку).
- Пул машин (≈10), неактивні переюзовуються.
- **Зіткнення (AABB):** удар, якщо `|car.z − player.z| < 2.6` і
  `|car.x − player.x| < 1.6`.
- При зіткненні: −1 життя, спалах екрана, тряска камери, haptic-вібро,
  невразливість ≈1.2 с (авто блимає), сусідній трафік відкидається далеко.

## Рівні

- Дистанція рівня накопичується = `speed * dt`.
- Ціль рівня: `LEVEL_TARGET(level) = 600 + level * 400` (метрів у межах рівня).
- При досягненні: `level++`, швидкість `+4`, інтервал спавну `−0.06`
  (мінімум 0.5 с), тост «РІВЕНЬ N», дистанція рівня скидається.
- Базова швидкість 26, базовий інтервал спавну 1.1 с.
- Загальний рахунок = сумарна пройдена дистанція (метри).

## Життя

- Старт: 3 життя (`❤❤❤` у HUD).
- Зіткнення: −1. При 0 — `gameover`.

## Вибір авто (стилі)

Кожен стиль: колір кузова + прапорці. Будуються процедурно (`buildCar`).

| id | назва | колір | особливість |
|------|----------|-----------|----------------------|
| taxi | Таксі | жовтий | дах-ліхтар + шашки |
| sport | Спорт | червоний | нижчий кузов |
| police | Поліція | синій | сині/червоні маячки |
| classic | Класика | чорний | — |
| lime | Лайм | зелений | — |

## Вибір міста (теми)

Кожна тема змінює фон/туман/колір будівель/палітру неону/амбієнт.

| id | назва | настрій |
|-------|-----------|----------------------------|

## Сцена (візуал, зберігається з 3D-демо)

- Процедурні хмарочоси з освітленими вікнами, туман глибини.
- Мокрий асфальт із відбиттями (Reflector) + смуги розмітки.
- Вуличні ліхтарі (теплі точкові світла), тротуари, дощ, неон-білборди.
- Постобробка: UnrealBloom (тільки яскраве цвіте).
- Камера-переслідувач за авто.

## Продуктивність (мобільний вебв'ю)

- `pixelRatio` ≤ 1.5; лише гравець має фари-spotlights; трафік — лише
  emissive-вогні; обмежена кількість динамічних світел; bloom threshold 0.6.

## Критерії готовності (що перевіряємо)

- [ ] `npm run typecheck` чистий, `npm run build` успішний.
- [ ] Меню показується; вибір авто/міста підсвічується.
- [ ] «Грати» стартує гру; HUD оновлює дистанцію/рівень/життя.
- [ ] Трафік спавниться й рухається; зіткнення забирає життя.
- [ ] Досягнення цілі → тост рівня, складність зростає.
- [ ] 0 життів → екран «Аварія»; «Ще раз»/«Меню» працюють.
- [ ] Зміна міста змінює палітру; зміна авто змінює модель.
- [ ] Рендер без помилок у headless-браузері (скриншоти).
